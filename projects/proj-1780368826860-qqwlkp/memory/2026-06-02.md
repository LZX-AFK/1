### 关键技术决策
1. **框架选型**：uni-app (Vue 3 + TypeScript + Pinia)，兼容 iOS/Android
2. **录音方案**：MVP 阶段使用手机麦克风 (`uni.getRecorderManager()`)，BLE 耳机预留接口
3. **WebSocket**：自封装 `ClassNoteWebSocket` 类，支持断线重连（指数退避 1s→2s→4s→...→30s max）、心跳保活、消息分发
4. **状态管理**：Pinia 拆分为 `session` 和 `recorder` 两个 store，职责清晰
5. **网络层**：`request.ts` 自动注入 Bearer Token，401 自动清除 token 并跳转首页
6. **暗色主题**：品牌色 `#6C63FF` 为主色调，深色背景 `#0F0F1A`

### 核心实现细节

#### buildSegments 时间分段算法
- `SEGMENT_DURATION_MS = 30_000`（30秒一段）
- 计算 maxMs = max(recordingDurationMs, 所有markers最大时间, 所有transcripts最大end时间)
- 每段包含：该时间窗口内的 final 转写文本 + 标记列表
- Tab 过滤：`filteredSegments` = 仅显示含有当前 tab 类型标记的段落（全部 tab 显示所有段）

#### InnerAudioContext 回放
- `onMounted` 初始化：仅当 `currentSession.audioUrl` 非空时创建
- `replaySegment(seg)` → `seekAndPlay(seg.startMs)` → `audioCtx.seek(sec).play()`
- MVP 降级：无音频文件时 Toast 提示时间戳，不崩溃
- 回放状态展示：isPlaying + pulse 动画绿点 + 当前时间 + 停止按钮
- `onUnmounted` 销毁 AudioContext 防止内存泄漏

#### 标记 Chip 设计
- 每个标记显示为彩色边框胶囊：颜色点 + 标签文字 + 时间戳
- 点击 chip → 底部弹窗（slide-up 动画）
- 弹窗内容：标记类型徽章 / 时间 / 笔记 / AI解析（Mock，按标签类型定制文案）/ 从此处回放

#### 颜色系统
- `didnt_understand` → `#FF6B6B`（accent红）
- `important` → `#FFC107`（warning橙黄）
- `exam_tip` → `#6C63FF`（primary紫）
- `question` → `#448AFF`（info蓝）
- `note` → `#00D9A6`（secondary绿）
