# 食愈（团团食记）后端开发方案

> **适用对象**：B（AI功能后端）、C（基础后端）  
> **版本**：v1.0  
> **最后更新**：2025-06-02  

---

## 一、分工总览

```
B ── F1 拍照识别+记录 ──→ F3 AI晚餐推荐 ──→ F5 食物衣柜
C ── F7 用户资料      ──→ F4 食谱详情    ──→ F2 今日汇总 ──→ F6 饮食日历
```

| 模块 | 负责人 | 核心工作 | 产出物 |
|------|--------|---------|--------|
| **F1** 拍照识别+记录 | B | 建 `meals` 表 + 3 个云函数 | 图片→识别→卡片→入库 全链路 |
| **F3** AI 晚餐推荐 | B | 写 `recommend` 云函数 | DeepSeek 生成推荐理由 |
| **F5** 食物衣柜 | B | 建 `skins` 表 + 预生成 20 张角色卡 + 解锁逻辑 | 衣柜系统后端 |
| **F7** 用户资料详情 | C | 建 `users` 表 + CRUD 云函数 | 用户设置读写 |
| **F4** 食谱详情 | C | 建 `recipes` 表 + 录入 30-50 条数据 + 查询云函数 | 食谱数据库 |
| **F2** 今日饮食汇总 | C | 写聚合查询云函数 | 今日热量/花费统计 |
| **F6** 饮食日历 | C | 写日期维度查询云函数 | 日历月视图+按日查询 |

## 二、数据库设计

### 2.1 B 负责建的表（3 张）

#### meals（食物记录表）

| 字段 | 类型 | 说明 | 来源 |
|------|------|------|------|
| `_id` | string | 自动生成 | CloudDB |
| `openid` | string | 用户标识 | 云函数自动获取 |
| `food_name` | string | 食物名称 | 百度 API 识别 |
| `calories` | number | 热量（kcal） | 百度 API + 份量换算 |
| `fat_ratio` | number | 脂肪占比 % | 百度 API |
| `protein_ratio` | number | 蛋白质占比 % | 百度 API |
| `carb_ratio` | number | 碳水占比 % | 百度 API |
| `category` | string | 食物大类 | 20 类映射 |
| `price` | number | 价格（元） | 用户手动填写 |
| `portion` | string | 份量 | 小份/中份/大份 |
| `image_url` | string | 食物原图 | 云存储 fileID（A 上传） |
| `skin_url` | string | chibi 皮肤图 | 豆包生图 → 云存储 fileID |
| `dialogue` | string | 拟人对话文案 | DeepSeek 生成 |
| `created_at` | date | 记录时间 | 云函数写入时间戳 |

#### skins（皮肤/角色卡表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `_id` | string | 自动生成 |
| `openid` | string | 用户标识 |

| `skin_url` | string | 角色卡图片 cloud://fileID |
| `card_name` | string | 卡片显示名（汉堡小人/面条小人...） |
| `unlocked_at` | date | 解锁时间 |

## 四、B 和 C 的交叉点

### 唯一的同步点：meals 表字段名

```
B Day1 上午：建好 meals 表 → 把字段名发微信群里

C 收到后：才能写 F2 的 getTodayMeals（要读 meals 表）
         才能写 F6 的 getMealsByDate/getMealsByMonth（要读 meals 表）
```

**其余全部独立，无第二个交叉点。**

### 云函数之间的调用关系

```
B 的云函数之间：不互相调用（A 前端串联）
C 的云函数之间：不互相调用（A 前端串联）
B 的云函数和 C 的云函数之间：不互相调用（A 前端串联）
```

**所有云函数都是独立原子操作，由前端 A 负责调用顺序和组合。**

### 数据流的串联方式

```
F1 记录流程：
  A → B.identify → A → B.generateSkinAndDialogue → A → B.addMeal

F2 汇总查询：
  A → C.getTodayMeals

F3 推荐流程：
  A → C.getTodayMeals（拿今日数据）→ A → B.recommend（传今日数据）→ A

F4 食谱详情：
  A → C.getRecipes（拿食谱列表）→ A

F5 衣柜：
  A → B.checkUnlock（判断解锁）→ A → B.getUserSkins（拿衣柜列表）

F6 日历：
  A → C.getMealsByMonth → A → C.getMealsByDate（点某天时）

F7 用户设置：
  A → C.getUser → A → C.saveUser
```

## 八、接口契约（给 A 前端用）

### B 的接口

```
wx.cloud.callFunction({ name: "identify", data: { imageFileID: "cloud://xxx" } })
→ { name, calories, fat_ratio, protein_ratio, carb_ratio, category, recognized }
  (recognized=false 时只有 name 为空，其余字段都不返回)

wx.cloud.callFunction({ name: "generateSkinAndDialogue", data: { food_name, calories, fat_ratio, protein_ratio, carb_ratio } })
→ { skin_url, dialogue, is_default_skin, is_default_dialogue }

wx.cloud.callFunction({ name: "addMeal", data: { food_name, calories, fat_ratio, protein_ratio, carb_ratio, category, price, portion, image_url, skin_url, dialogue } })
→ { success, meal_id, leaf_earned }

wx.cloud.callFunction({ name: "recommend", data: { todays_meals: [...], remaining_calories: 500, goal: "lose_fat" } })
→ { recommendations: [{ name, calories, reason, recipe_id, type }] }

wx.cloud.callFunction({ name: "checkUnlock", data: { food_category: "burger" } })
→ { is_new, skin_url, card_name }

wx.cloud.callFunction({ name: "getUserSkins" })
→ { skins: [{ food_category, skin_url, card_name, unlocked_at }], unlocked_count: 3, total_count: 20 }
```

### C 的接口

```
wx.cloud.callFunction({ name: "saveUser", data: { height: 164, weight: 56, age: 24, goal: "lose_fat", budget: 50 } })
→ { success }

wx.cloud.callFunction({ name: "getUser" })
→ { user: { height, weight, age, goal, budget, ... } }

wx.cloud.callFunction({ name: "getTodayMeals" })
```
